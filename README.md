# Лабораторная 1

_Nota:_ Dockerfile'ы лежат в `app/`. Плохой: `Dockerfile.bad`. Хороший: `Dockerfile`.

## Плохие практики при написании Dockerfile'ов

- **Не использование multi stage building**
  - **Почему плохо:** образ раздувается (образ по хорошему файлу: **183Mb**. По плохому: **~900Mb**)
- **Делать все под root'ом**
  - **Почему плохо:** небезопасно
- **Не разделять `ENTRYPOINT` и `CMD`**
  - **Почему плохо:** `ENTRYPOINT` - не изменяемая команда, а `CMD` - аргументы по умолчанию, значит можно запустить с кастомными флагами    

    
## Плохие практики при работе с Docker контейнерами
- **Хранить чувствительные данные в контейнерах**
  - **Почему плохо:** при обновлении контейнера все данные потеряются
  - **Что делать:** использовать `volumes`
- **Запускать контейнеры без ограничения на потребление памяти**
  - **Почему плохо:** контейнер может потреблять слишком много памяти
  - **Что делать:** `docker run --memory="512m" --cpus="1.0" myapp`
